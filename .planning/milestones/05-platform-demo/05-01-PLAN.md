# Plan 05-01: Demo Reliability & Health Check

## Overview

**Goal**: Ensure the application handles failures gracefully and provides pre-demo validation tools.

**Requirements**: DEMO-01, DEMO-02, DEMO-03

**Dependencies**: Phase 2 (Backend), Phase 3 (ML)

## Success Criteria

1. Application displays fallback data and clear error state when API fails
2. Health check script validates all services (backend, frontend, predictions)
3. Warm-up script pre-loads model serving endpoints to avoid cold-start delays
4. UI shows clear connection status and error messages

## Tasks

### Task 1: Backend Error Handling & Fallback

**Files to modify:**
- `app/backend/services/flight_service.py` - Add robust fallback when data unavailable
- `app/backend/api/routes.py` - Return graceful error responses with fallback data
- `app/backend/api/predictions.py` - Handle model serving unavailability

**Implementation:**
1. Wrap data fetching with try/catch and return cached/synthetic data on failure
2. Add `data_source` field to responses ("live", "cached", "synthetic")
3. Predictions return "unavailable" with reason instead of 500 errors
4. Add configurable fallback mode via environment variable

### Task 2: Frontend Error State UI

**Files to modify:**
- `app/frontend/src/hooks/useFlights.ts` - Better error handling
- `app/frontend/src/components/Header/Header.tsx` - Show data source indicator
- `app/frontend/src/App.tsx` - Global error boundary

**Implementation:**
1. Display banner when using fallback data ("Demo mode - using cached data")
2. Show connection status in header (green/yellow/red indicator)
3. Toast notifications for transient errors
4. Error boundary prevents white screen crashes

### Task 3: Pre-Demo Health Check Script

**Files to create:**
- `scripts/health_check.py` - Validates all services

**Implementation:**
```python
# Check endpoints:
# - GET /health - Backend alive
# - GET /api/flights - Data flowing
# - GET /api/predictions/delay/{icao24} - Predictions working
# - WebSocket connection test

# Output: JSON report with status per service
# Exit code: 0 if all pass, 1 if any fail
```

### Task 4: Service Warm-up Script

**Files to create:**
- `scripts/warmup.py` - Pre-warms services before demo

**Implementation:**
1. Make N requests to each endpoint to warm caches
2. Trigger prediction endpoints to load models
3. Verify response times are within thresholds
4. Output timing report

## UAT Criteria

- [ ] When backend cannot fetch live data, UI shows synthetic flights with "Demo Mode" banner
- [ ] Health check script reports status of all services in JSON format
- [ ] Warmup script completes within 30 seconds and reports timing
- [ ] No white screen errors when services fail - always graceful degradation
