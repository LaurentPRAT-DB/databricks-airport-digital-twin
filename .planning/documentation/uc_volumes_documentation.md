# Plan: Document UC Volume Static Assets and Deployment Process

## Context

The project uses 5 Unity Catalog Managed Volumes to store static assets, calibration data, simulation outputs, and collected data. These are declared via DABs but populated through separate scripts/jobs. There's no single document that inventories all assets, their lifecycle, and how they get deployed. This documentation will help new contributors and future maintenance.

## Target File

Create `docs/UC_VOLUMES.md` — a comprehensive reference document covering all volumes.

## Volume Inventory (from research)

### 1. calibration_profiles (5.8 MB, 1,183 JSON files)

- **Resource:** `resources/calibration_profiles_volume.yml`
- **Contents:** Per-airport calibration JSON files (`{ICAO}.json`) — traffic stats, fleet mix, route distributions derived from BTS OTP data
- **Local source:** `data/calibration/profiles/*.json`
- **How populated:**
  - Initial: `profile_sync` job (`databricks/notebooks/sync_profiles_to_uc.py`) — one-time bulk MERGE into UC table
  - Ongoing: `calibration_refresh` job (weekly cron, `databricks/notebooks/build_calibration_profiles.py`) — re-derives from BTS data
  - Manual: `scripts/build_airport_profiles.py` builds locally, then `_persist_to_unity_catalog()` uploads
- **Runtime read:** `src/calibration/profile.py` → `_load_from_volume()` reads `/Volumes/{cat}/{sch}/calibration_profiles/{ICAO}.json`
- **Fallback chain:** UC table → UC Volume → local JSON → known_profiles → OpenFlights → hardcoded

### 2. static_assets (57 MB 3D models + airport cache JSONs)

- **Resource:** `resources/static_assets_volume.yml`
- **Contents:**
  - `models/` — 16 GLB 3D aircraft models (airbus, boeing, fighters)
  - `airport_cache/airport_{ICAO}.json` — pre-computed OSM airport geometry (26 airports)
- **Local source:** `app/frontend/dist/models/aircraft/*.glb` (models), no local source for airport_cache (generated by job)
- **How populated:**
  - 3D models: Manual upload (`databricks fs cp`) — done once, rarely changes
  - Airport cache: `osm_preload` job (`databricks/notebooks/preload_osm_airports.py`) — fetches from Overpass API and persists to UC tables (not directly to volume). Backend `_prewarm_airports_background()` seeds Lakebase from `static_assets/airport_cache/` at startup
- **Runtime read:** `app/backend/main.py` serves `/models/*` from UC Volume path, and reads `airport_cache/` for prewarm

### 3. simulation_data (variable size — grows with usage)

- **Resource:** `resources/simulation_data_volume.yml`
- **Contents:**
  - `*.json` — saved simulation output files (flight positions, events)
  - `debug/` — runtime debug logs (`client_debug.log`)
  - `ml_models/` — trained model pickles (`obt_refined.pkl`, `turnaround_model.pkl`)
  - `msfs_scenery/` — MSFS BGL scenery files
  - Simulation reports and metadata
- **Local source:** `data/ml_models/` (2 pickle files, 880K)
- **How populated:**
  - Sim outputs: Written by simulation batch jobs (`run_simulation_airport.py`, `run_all_simulations.py`)
  - ML models: Written by training jobs (`train_obt_model.py`, `train_turnaround_model.py`)
  - MSFS: One-time upload via `databricks/notebooks/persist_msfs_bgl.py`
  - Debug: Written at runtime by the app backend
- **Runtime read:** `app/backend/api/simulation.py` loads sim files and reports; `src/ml/registry.py` loads models

### 4. demo_simulations (pre-generated demo data)

- **Resource:** `resources/demo_simulations_volume.yml`
- **Contents:** `demo_{ICAO}.json` — pre-generated 24h simulation for default airports
- **Local source:** `data/demo/` (currently empty — generated on Databricks)
- **How populated:**
  - Generated by simulation batch jobs, then manually copied to this volume
  - Or generated on-demand by `DemoSimulationService` and cached
- **Runtime read:** `app/backend/services/demo_simulation_service.py` → `_download_from_volume()` downloads on airport switch

### 5. opensky_raw (92 MB JSONL files)

- **Resource:** `resources/opensky_raw_volume.yml`
- **Contents:** JSONL files with raw ADS-B state vectors from OpenSky Network API
- **Local source:** `data/opensky_raw/*.jsonl` (collected locally, moved to `synced/` after upload)
- **How populated:**
  - `scripts/sync_opensky_to_volume.sh` — chunked upload (batch of 100), triggers ingestion job
  - Manual: `databricks fs cp data/opensky_raw/*.jsonl dbfs:/Volumes/.../opensky_raw/`
- **Runtime read:** Not read by app directly — consumed by `opensky_ingestion` job which processes into Delta tables

## Deployment Lifecycle

```
┌─────────────────────────────────────────────────────────────────┐
│  ./deploy.sh                                                     │
│  Step 2: databricks bundle deploy                                │
│    → Creates all 5 volumes (MANAGED, empty)                      │
│    → No data is uploaded during bundle deploy                    │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  Post-deploy: One-time data seeding (manual)                     │
│                                                                   │
│  1. Calibration profiles:                                        │
│     databricks bundle run profile_sync --target dev              │
│                                                                   │
│  2. OSM airport cache:                                           │
│     databricks bundle run osm_preload --target dev               │
│                                                                   │
│  3. 3D models:                                                   │
│     databricks fs cp -r app/frontend/dist/models/aircraft/ \     │
│       dbfs:/Volumes/.../static_assets/models/                    │
│                                                                   │
│  4. OpenSky raw data:                                            │
│     ./scripts/sync_opensky_to_volume.sh                          │
│                                                                   │
│  5. Demo simulations:                                            │
│     databricks bundle run simulation_batch_1 --target dev        │
│     (then copy outputs to demo_simulations volume)               │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  Ongoing: Scheduled jobs                                         │
│                                                                   │
│  • calibration_refresh (weekly MON 06:00 UTC)                    │
│  • opensky_ingestion (triggered after sync_opensky_to_volume.sh) │
│  • simulation batches (on-demand or scheduled)                   │
│  • obt_model_training / turnaround training (on-demand)          │
└─────────────────────────────────────────────────────────────────┘
```

## What to Document

The new `docs/UC_VOLUMES.md` will contain:

1. **Overview table** — volume name, purpose, size, population method
2. **Per-volume sections** — contents, local source, upload commands, runtime usage, fallback behavior
3. **First-time setup** — ordered steps to seed volumes after a fresh deploy
4. **Ongoing maintenance** — scheduled jobs that keep volumes up to date
5. **Permissions** — what grants the app SP needs (`READ VOLUME`)
