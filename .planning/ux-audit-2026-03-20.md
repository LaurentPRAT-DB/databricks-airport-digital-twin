# UX Audit Report — Airport Digital Twin
**Date:** 2026-03-20
**App:** https://airport-digital-twin-dev-7474645572615955.aws.databricksapps.com
**Build:** v0.1.0, #268
**Airports tested:** KSFO, RJTT (HND), EGLL (LHR)

---

## Executive Summary

The application provides a compelling, functional airport digital twin with real-time flight simulation, gate management, turnaround tracking, and multi-airport support. The core experience is solid — flights move realistically, turnaround phases progress logically, and the 2D/3D views are coherent. However, several issues reduce realism and polish, primarily around airline/route authenticity in uncalibrated mode and airport switching latency.

---

## 1. 2D View — Trajectories, Taxi, Takeoff

### What works well
- Aircraft icons render with correct heading rotation
- Flight list shows 100 flights with realistic airline mix for KSFO (UAL dominant, ASA, DAL, SWA, AAL present)
- Descending flights show decreasing altitude over time (DAL1245: 1997ft → 1899ft → lower)
- Taxiing aircraft move at realistic 25kts ground speed
- Flight trajectory visualization with "Show Trajectory" button (80 data points)
- Satellite tile layer renders clearly

### Issues found
| ID | Severity | Issue | Details |
|----|----------|-------|---------|
| 2D-1 | Medium | Unrealistic routes in uncalibrated mode | Ryanair SFO→Cambridge, Southwest at Tokyo/London — the 1,180-airport lookup table provides valid airports but doesn't filter by airline route networks |
| 2D-2 | Low | Fixed weather data | "15°C 279@10kt 10SM" doesn't change between airports or over time |
| 2D-3 | Low | Trajectory tooltip limited | Only shows point count, no visual of approach/departure path phases |

## 2. Flight Details Panel

### What works well
- Comprehensive data: origin/dest, aircraft type, lat/lon, altitude, speed, heading, vertical rate
- Delay prediction with confidence percentage (e.g., "Slight Delay +10m 80%")
- Baggage status with delivery %, total count, connecting bags
- Carousel assignment shown
- Data source and last-seen timestamp

### Issues found
| ID | Severity | Issue | Details |
|----|----------|-------|---------|
| FD-1 | Medium | "Departing to" field blank | ANA5242 at Gate G3 shows "SFO" under "Arrived from" and blank "Departing to" — destination should be assigned for next leg |
| FD-2 | Low | ICAO hex code placeholder | UAL123 shows ICAO "a12345" — should be a realistic hex transponder code |

## 3. Gate Turnaround & Flight Cycle

### What works well
- 7-phase turnaround pipeline: deboarding → unloading → cleaning → catering → boarding → loading → pushback
- Checkmarks (✓) show completed phases, numbered circles for pending
- Progress percentage updates in real-time (ANA5242: 15% → 17% over ~30s)
- Active equipment displayed (belt loader x2, ground power)
- Estimated departure time shown
- Gate occupancy tracked per terminal (e.g., "Terminal A: 9 free | 6 used / 15")

### Issues found
| ID | Severity | Issue | Details |
|----|----------|-------|---------|
| GT-1 | **High** | Speed 25kts during gate turnaround | UAL123 at Gate A4 in "Cleaning" phase at 62% shows speed=25kts and heading changing — aircraft should be stationary (0kts) while at gate |
| GT-2 | Medium | Baggage total = 0 after delivery complete | UAL123 shows "136 Total, 0 Delivered" but "100% Delivered" — contradictory display. Likely the delivered count field shows 0 when all delivered vs actual count |
| GT-3 | Low | Gate congestion always "Low" | Terminal congestion indicator never shows Moderate/High/Critical despite 50+ occupied gates |

## 4. Airport Switching

### What works well
- Airport selector dropdown well-organized by region (Americas, Europe, Middle East, Asia-Pacific, Africa)
- 29 pre-configured airports with ICAO/IATA codes, names, and locations
- Custom ICAO input field for any airport
- "All 29 airports cached" indicator
- Flight list fully refreshes with new flights on airport switch
- Gate status panel updates with correct terminal structure per airport
- Current airport shown in header with ICAO (IATA) format

### Timing measurements
| Switch | Time | Notes |
|--------|------|-------|
| KSFO → RJTT | ~21s | Full OSM fetch, no progress overlay shown |
| RJTT → EGLL | ~13s | Possibly partially cached |

### Issues found
| ID | Severity | Issue | Details |
|----|----------|-------|---------|
| AS-1 | **High** | 13-21s switch time with no progress feedback | No loading spinner or progress overlay visible during airport switch — user sees stale airport while new one loads |
| AS-2 | Medium | Terminal mapping wrong for some airports | EGLL shows Terminals 2, 3, 4, 6, A, B, C — Heathrow actually has T2, T3, T4, T5 (no A/B/C). OSM data mapping issue |
| AS-3 | Medium | RJTT: Terminal 9 "0 free, 1 used / 1" | Single-gate terminals suggest OSM data quality issue for some airports |
| AS-4 | Medium | Stale terminal tabs from previous airport | After switching RJTT → EGLL, Terminal tabs "2", "4", "6" appear to carry over from RJTT numbering |

### What works well
- Airport 3D geometry renders correctly (terminals as extruded buildings, runways, taxiways)
- Aircraft 3D models positioned on the ground/in air appropriately
- Green flight labels float above aircraft with callsign, altitude, speed

### 2D/3D Coherence
- **Positions match:** BAW308 at (51.4875, -0.4541) in both views ✓
- **Flight list consistent** between views ✓
- **Gate status panel identical** ✓
- **View switch time:** < 1 second ✓

### Issues found
| ID | Severity | Issue | Details |
|----|----------|-------|---------|
| 3D-1 | Low | No aircraft model variety | All aircraft appear to use same 3D model regardless of type (B737 vs A380 vs regional) |
| 3D-2 | Low | Flight count discrepancy | 2D status bar shows "Flights: 149" but header shows "Flights: 100" — the status bar may include newly spawned flights before the counter updates |

## 6. Airline Realism (Cross-cutting)

| Airport | Realistic airlines present | Unrealistic airlines present |
|---------|--------------------------|------------------------------|
| KSFO | UAL (dominant), ASA, DAL, SWA, AAL, ANA, JAL, KLM, CZ, AMX | EDV, FFT, PDT at SFO are plausible as regional |
| RJTT | ANA, BAW, CPA, UAE | DAL (many), AAL (many), SWA, JBU, ASA — impossible at Tokyo Haneda |
| EGLL | BAW (dominant ~40%), VIR, CPA, ACA, AFR, DLH, SIA, TAP, THY, UAE, QFA, ANA | SWA789 — impossible at Heathrow |

**Root cause:** Without calibration data, the flight generator picks airlines from a generic pool. The calibration system (33 known profiles) fixes this for calibrated airports but uncalibrated airports get the default US-centric mix.

## Improvement Plan (Prioritized)

### P0 — Critical (fix now)
1. **Fix speed during turnaround** (GT-1): Aircraft at gate during turnaround phases must show 0kts speed and fixed heading. The position/speed update should check turnaround state.
2. **Add airport switch loading indicator** (AS-1): The loading spinner/progress overlay that was implemented should be showing during airport switches. Verify `isLoading` state is properly set during the OSM fetch.

### P1 — Important (next sprint)
3. **Per-airport airline profiles for uncalibrated airports** (2D-1): Create regional airline pools (e.g., Japan → ANA/JAL/APJ/SFJ dominant, UK → BAW/VIR/EZY dominant) so even uncalibrated airports get plausible airline mixes.
4. **Remove "fixed" flights from all airports** (SWA789, DAL456, UAL123, AAL100, JBU555): These appear at every airport regardless of realism — they should be suppressed or generated airport-aware.
5. **Fix terminal mapping from OSM** (AS-2): Some OSM terminal names don't match real airport terminal numbering. Add validation/override logic for well-known airports.
6. **Fix baggage display contradiction** (GT-2): When 100% delivered, show actual delivered count, not 0.

### P2 — Polish (backlog)
7. **Weather per airport**: Fetch real METAR data or at least randomize weather per airport instead of showing fixed "15°C 279@10kt 10SM" everywhere.
8. **Gate congestion thresholds** (GT-3): Tune congestion calculation so it actually triggers Moderate/High/Critical at realistic occupancy levels.
9. **Aircraft model variety in 3D** (3D-1): Map aircraft type (B737, A320, B777, A380) to different 3D models.
10. **Realistic transponder hex codes** (FD-2): Generate plausible Mode-S hex codes instead of "a12345".
11. **"Departing to" always populated** (FD-1): Ensure next destination is assigned as soon as turnaround begins.
12. **Fix flight count discrepancy** (3D-2): Ensure header counter and status bar agree on flight count.
13. **Clean up stale terminal tabs on airport switch** (AS-4): Terminal tabs from previous airport should be fully cleared.

### P3 — Future enhancement
14. **Pre-load airport data**: Cache OSM data for all 29 airports at startup to eliminate switch latency (plan exists at `.planning/phases/13-airport-preload-cache/PLAN.md`).
15. **Airport switch animation**: Smooth map transition between airports (fly-to animation).
16. **2D/3D state sync**: Share selection and zoom level between 2D and 3D views.
