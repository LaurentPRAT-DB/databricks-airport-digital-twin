# UX Review — Airport Digital Twin (Updated)

**Date:** 2026-03-21 (rev 2)
**Reviewer:** Claude (acting as UX designer)
**App URL:** https://airport-digital-twin-dev-7474645572615955.aws.databricksapps.com
**Airports tested:** RJTT (Tokyo Haneda), KSFO (San Francisco), EGLL (London Heathrow)

---

## Executive Summary

Since the initial review, significant progress has been made: **departures now work** (climbing flights observed), **FIDS is partially connected** to live data, and **3D aircraft lighting has been improved**. The simulation now runs a full arrival-to-departure lifecycle. However, new priorities have emerged: **FIDS routes are sometimes unrealistic** (showing random destinations for uncalibrated airports), **airport switching is still slow** (17-19s), and **gate positioning precision** could be improved. The OpenFlights route ingestion system has been implemented but profiles need to be built for all airports.

**Overall UX Score: 8/10** — Strong simulation with full lifecycle, excellent airline mixes, and stable operation. Route realism and loading performance are the main gaps.

---

## 1. 2D View

### What Works Well
- **OSM overlay accuracy:** Terminal buildings, taxiways, aprons, and runways align precisely with satellite imagery across all 3 tested airports
- **Aircraft silhouette sizing:** Proportionally correct relative to terminal buildings at all zoom levels
- **Gate labels:** Clear, readable gate numbers at correct positions along terminal buildings
- **Flight detail panel:** Rich — callsign, state, origin/destination, aircraft type, speed, heading, delay prediction with confidence %, turnaround progress, baggage status, active GSE
- **Trajectory visualization:** Approach paths curve from origin bearing, departures aim toward destination. Altitude-colored polylines are intuitive
- **Airline mix realism:** Excellent calibration:
  - RJTT: ANA ~45, JAL ~38, SKY ~8, SFJ ~5, ADO ~5 — accurate for Haneda
  - KSFO: UAL, SWA, DAL, AAL, BAW — accurate for SFO
  - EGLL: BAW dominant (~60%) — accurate for Heathrow
- **Full flight lifecycle:** Approach -> descend -> taxi -> park -> turnaround -> pushback -> taxi-out -> depart -> climb all observed and working
- **Satellite view toggle:** Clean switch with OSM overlay preserved

### Issues Found

| # | Severity | Issue | Details |
|---|----------|-------|---------|
| 2D-1 | **P1** | Non-directional aircraft markers | 2D silhouettes are simple dots/icons without heading rotation. Cannot tell which way an aircraft is facing at a gate. Directional arrow or oriented silhouette would improve realism. |
| 2D-2 | **P1** | Gate label clutter at medium zoom | At medium zoom levels, gate labels overlap and become unreadable. Labels should hide at low zoom and only appear when zoomed in close. |
| 2D-3 | **P2** | Gate-less ground flights | A few flights on ground with `assigned_gate: null` appear parked at arbitrary coordinates, not on taxiways. |

### Issues Found

| # | Severity | Issue | Details |

## 5. Gate Operations & Flight Lifecycle

### What Works Well
- **Full lifecycle working:** Arrivals descend, taxi, park at gates. After turnaround, flights pushback, taxi out, and depart with climbing trajectory. This is a major improvement from previous review.
- **Turnaround progress system:** 7-phase turnaround with real-time percentage advancement
- **Active GSE tracking:** Belt loaders, ground power, etc. per gate
- **Baggage status:** Detailed counts — total, delivered, connecting, at-risk bags
- **Delay prediction:** ML-based forecast with confidence %
- **Gate occupancy tracking:** Real-time available/occupied counts per terminal

### Issues

| # | Severity | Issue | Details |
|---|----------|-------|---------|
| GL-1 | **P1** | Gate nose-in precision | Aircraft at gates are near terminals but not always precisely nose-in to the terminal wall with wings parallel. Some appear broadside. |
| GL-2 | **P2** | Turnaround timing not calibrated | Turnaround rate doesn't align with estimated departure time. Progress may advance too slowly or quickly relative to scheduled departure. |

## 8. Improvement Plan (Prioritized)

### P0 — Critical (Demo Quality)

#### 1. Build OpenFlights profiles for all airports -- DONE
- Ran `python scripts/build_airport_profiles.py --source openflights --all-airports`
- Downloaded `routes.dat` (67,663 routes), built 1,180 profiles (1,062 OpenFlights + 118 fallback)
- Deployed: all profile JSONs included in DABs bundle via `data/calibration/profiles/**`
- **Result:** RJTT shows FUK/CTS/KIX domestic, LHR/TSA/HKG intl. EGLL shows JFK/LAX/ORD top routes.

#### 2. Fix gate label zoom clutter -- DONE
- Raised `GATE_LABEL_ZOOM` from 16 to 17 in `AirportOverlay.tsx`
- Labels now only show as permanent text at zoom 17+, reducing overlap at medium zoom
- Hover tooltips still available at all zoom levels
- Test updated in `AirportOverlay.test.tsx`

### P1 — High Priority

#### 3. Directional 2D aircraft markers -- ALREADY DONE
- `FlightMarker.tsx` already applies `rotate(${heading}deg)` CSS transform to SVG silhouettes
- Aircraft silhouettes are category-specific (narrow/wideTwin/wideQuad) with geo-realistic sizing
- No changes needed — this was a false finding in the review

#### 4. Improve gate nose-in precision -- ALREADY DONE
- `_get_parked_heading()` in `fallback.py:2266` already computes heading perpendicular to nearest terminal edge
- Uses OSM terminal polygon geometry, finds nearest edge, computes inward-facing normal
- Falls back to airport center bearing if no terminal data available
- No changes needed — the algorithm is already sophisticated

#### 5. Pre-cache airports for faster switching -- DONE
- Added `_prewarm_airports_background()` to `main.py` that runs after startup
- Iterates all 29 well-known airports, loads via `initialize_from_lakehouse()` into Lakebase
- Uses a fresh `AirportConfigService` instance so active airport config isn't clobbered
- 2s delay between OSM requests to be polite to Overpass API
- After first run, all airports are Tier 1 cache hits (~0.3-0.5s config load)

### P2 — Medium Priority

#### 6. 2D/3D selection sync
- Clicking a flight in 2D should highlight it in 3D and vice versa
- **Effort:** Medium

#### 7. 3D ground plane realism
- Add runway markings and taxiway lines
- Basic apron/concrete texture
- **Effort:** Medium

#### 8. 3D flight labels
- Show callsign labels on hover or toggle in 3D view
- **Effort:** Small

#### 9. Calibrate turnaround timing
- Align turnaround progress rate with estimated departure time
- Realistic phase durations (deboarding 10-20min, refueling 15-25min, etc.)
- **Effort:** Small

### P3 — Nice to Have

10. **3D camera reset improvement** — Default to mid-altitude terminal view
11. **Airport switch progress details** — Show stage: "Fetching OSM data... Generating flights..."
12. **Fix gate-less ground flights** — Position on taxiways or assign to gates
13. **Aircraft clustering avoidance** — Prevent overlap at busy gate areas
