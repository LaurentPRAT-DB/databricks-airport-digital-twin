# Unity Catalog Volumes — Static Assets Reference

All file-based assets in this project are stored in **Unity Catalog Managed Volumes** under  
`serverless_stable_3n0ihb_catalog.airport_digital_twin`.

Volumes are **created empty** by `databricks bundle deploy` (DABs). Data is seeded separately via jobs or manual upload.

---

## Volume Inventory

| Volume | Purpose | Size | Population | Runtime Access |
|--------|---------|------|------------|----------------|
| `calibration_profiles` | Per-airport traffic/fleet statistics | 5.8 MB (1,183 files) | `profile_sync` job + weekly refresh | Backend reads on airport load |
| `static_assets` | 3D models + pre-computed OSM cache | ~57 MB | Manual upload + `osm_preload` job | Backend serves `/models/*` and seeds Lakebase |
| `simulation_data` | Sim outputs, ML models, debug logs | Variable (grows) | Batch jobs + runtime writes | Backend loads sim files and ML pickles |
| `demo_simulations` | Pre-generated demo replays | ~50 MB per airport | Batch jobs (then copy) | Backend auto-downloads on airport switch |
| `opensky_raw` | Raw ADS-B JSONL from OpenSky Network | 92 MB+ | `sync_opensky_to_volume.sh` | Not read by app — consumed by ingestion job |

## 1. `calibration_profiles`

**Resource config:** `resources/calibration_profiles_volume.yml`

**Contents:**  
Per-airport JSON files named `{ICAO}.json` containing traffic statistics, fleet mix, airline shares, and route distributions derived from BTS On-Time Performance data.

**Volume path:** `/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/calibration_profiles/`

**Local source:** `data/calibration/profiles/*.json`

### How it gets populated

| Method | Command | When |
|--------|---------|------|
| Initial bulk sync | `databricks bundle run profile_sync --target dev` | After first deploy |
| Weekly refresh | `calibration_refresh` job (cron: MON 06:00 UTC) | Automatic |
| Manual rebuild | `python scripts/build_airport_profiles.py` | Ad-hoc |

### Runtime read path

```
src/calibration/profile.py → _load_from_volume(icao)
  reads: /Volumes/{catalog}/{schema}/calibration_profiles/{ICAO}.json
```

**Fallback chain:** UC `airport_profiles` table → UC Volume → local `data/calibration/profiles/` → `known_profiles.py` → OpenFlights → hardcoded defaults.

## 2. `static_assets`

**Resource config:** `resources/static_assets_volume.yml`

**Contents:**

| Subdirectory | Files | Description |
|--------------|-------|-------------|
| `models/` | 16 GLB files | 3D aircraft models (Airbus, Boeing, fighters) |
| `airport_cache/` | 26 JSON files | Pre-computed OSM airport geometry (`airport_{ICAO}.json`) |

**Volume path:** `/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/static_assets/`

**Local source:** `app/frontend/dist/models/aircraft/*.glb` (3D models only — airport cache is generated by job)

### How it gets populated

| Subdirectory | Method | Command |
|--------------|--------|---------|
| `models/` | Manual one-time upload | `databricks fs cp -r app/frontend/dist/models/aircraft/ dbfs:/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/static_assets/models/` |
| `airport_cache/` | `osm_preload` job | `databricks bundle run osm_preload --target dev` |

### Runtime read path

```python
# 3D models — served at /models/{subpath}
app/backend/main.py:
  UC_VOLUMES_MODEL_PATH = "/Volumes/.../static_assets/models"
  w.files.download(f"{UC_VOLUMES_MODEL_PATH}/{subpath}")

# Airport cache — bulk prewarm at startup
app/backend/main.py → _prewarm_airports_background():
  volume_base = f"/Volumes/{catalog}/{schema}/static_assets/airport_cache"
  w.files.download(f"{volume_base}/airport_{icao}.json")
```

**Fallback:** 3D models fall back to local `app/frontend/dist/models/`. Airport cache falls back to live OSM Overpass API fetch.

## 3. `simulation_data`

**Resource config:** `resources/simulation_data_volume.yml`

**Contents:**

| Subdirectory | Files | Description |
|--------------|-------|-------------|
| `*.json` | Variable | Saved simulation output (flight positions, events, metadata) |
| `ml_models/` | 2+ pickle files | Trained ML models (`obt_refined.pkl`, `turnaround_model.pkl`) |
| `debug/` | Log files | Runtime debug logs (`client_debug.log`) |
| `msfs_scenery/` | BGL files | Microsoft Flight Simulator scenery data |

**Volume path:** `/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/simulation_data/`

**Local source:** `data/ml_models/` (880K, 2 pickle files)

### How it gets populated

| Content | Method | Trigger |
|---------|--------|---------|
| Simulation outputs | Batch jobs write JSON | `databricks bundle run simulation_batch_1` |
| ML models | Training jobs write pickles | `databricks bundle run obt_model_training` |
| MSFS scenery | One-time notebook | `databricks/notebooks/persist_msfs_bgl.py` |
| Debug logs | Runtime | App backend writes on debug POST |

### Runtime read path

```python
# Simulation files
app/backend/api/simulation.py:
  volume_path = f"/Volumes/{catalog}/{schema}/simulation_data/{filename}"
  w.files.download(volume_path)

# ML models
src/ml/registry.py:
  _ML_MODELS_DIR = Path(f"/Volumes/{catalog}/{schema}/simulation_data/ml_models")

# Debug logs
app/backend/api/routes_debug.py:
  debug_dir = f"/Volumes/{catalog}/{schema}/simulation_data/debug"
```

## 4. `demo_simulations`

**Resource config:** `resources/demo_simulations_volume.yml`

**Contents:**  
Pre-generated 24h simulation files named `demo_{ICAO}.json` for well-known airports. Used for immediate replay on airport switch without running a live simulation.

**Volume path:** `/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/demo_simulations/`

**Local source:** `data/demo/` (empty locally — generated on Databricks)

### How it gets populated

1. Run simulation batch jobs for target airports
2. Copy output to demo volume:
   ```bash
   databricks fs cp simulation_output_KSFO.json \
     dbfs:/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/demo_simulations/demo_KSFO.json
   ```

Or the `DemoSimulationService` generates on-demand and caches the result.

### Runtime read path

```python
# app/backend/services/demo_simulation_service.py
vol = f"/Volumes/{catalog}/{schema}/demo_simulations"
remote_path = f"{vol}/demo_{airport_icao}.json"
w.files.download(remote_path)
```

**Fallback:** If not in volume, generates a shorter 6h simulation on demand.

## 5. `opensky_raw`

**Resource config:** `resources/opensky_raw_volume.yml`

**Contents:**  
JSONL files containing raw ADS-B state vectors collected from the OpenSky Network REST API. Each file covers one collection session.

**Volume path:** `/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/opensky_raw/`

**Local source:** `data/opensky_raw/*.jsonl` (moved to `data/opensky_raw/synced/` after upload)

### How it gets populated

```bash
# Chunked upload with automatic ingestion trigger
./scripts/sync_opensky_to_volume.sh              # batch of 100
./scripts/sync_opensky_to_volume.sh --batch 50   # smaller batches
./scripts/sync_opensky_to_volume.sh --dry-run    # preview

# Manual single file
databricks fs cp data/opensky_raw/session_20260507.jsonl \
  dbfs:/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/opensky_raw/
```

### Runtime read path

**Not read by the app.** Consumed by the `opensky_ingestion` job which processes JSONL into Delta tables for analytics.

## First-Time Setup (after fresh deploy)

Run these steps in order after `./deploy.sh` completes:

```bash
# 1. Seed calibration profiles (required for realistic simulations)
databricks bundle run profile_sync --target dev

# 2. Pre-load OSM airport geometry (avoids Overpass API calls at runtime)
databricks bundle run osm_preload --target dev

# 3. Upload 3D aircraft models (required for 3D view)
databricks fs cp -r app/frontend/dist/models/aircraft/ \
  dbfs:/Volumes/serverless_stable_3n0ihb_catalog/airport_digital_twin/static_assets/models/ \
  --overwrite

# 4. (Optional) Upload OpenSky raw data for enrichment pipeline
./scripts/sync_opensky_to_volume.sh

# 5. (Optional) Generate demo simulations for instant replay
databricks bundle run simulation_batch_1 --target dev
# Then copy outputs to demo_simulations volume
```

Steps 1-3 are required for full functionality. Steps 4-5 are optional (the app works without them via fallbacks).

## Ongoing Maintenance

| Job | Schedule | What it does |
|-----|----------|--------------|
| `calibration_refresh` | Weekly (MON 06:00 UTC) | Re-downloads BTS data, rebuilds all 1,183 profiles |
| `osm_preload` | On-demand | Re-fetches OSM geometry for all 26 airports |
| `opensky_ingestion` | Triggered after sync script | Processes JSONL into Delta enrichment tables |
| `obt_model_training` | On-demand | Retrains OBT model, writes pickle to `simulation_data/ml_models/` |
| `simulation_batch_*` | On-demand | Runs calibrated simulations, writes to `simulation_data/` |

## Permissions

The app Service Principal needs these grants (managed by `scripts/grant_sp_permissions.sh`):

```sql
GRANT READ VOLUME ON VOLUME serverless_stable_3n0ihb_catalog.airport_digital_twin.calibration_profiles TO `app-SP`;
GRANT READ VOLUME ON VOLUME serverless_stable_3n0ihb_catalog.airport_digital_twin.static_assets TO `app-SP`;
GRANT READ VOLUME, WRITE VOLUME ON VOLUME serverless_stable_3n0ihb_catalog.airport_digital_twin.simulation_data TO `app-SP`;
GRANT READ VOLUME ON VOLUME serverless_stable_3n0ihb_catalog.airport_digital_twin.demo_simulations TO `app-SP`;
-- opensky_raw: no app access needed (consumed by jobs only)
```
